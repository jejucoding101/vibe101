# 📌 26강: 프로젝트 기획 — 아이디어를 현실로

> **핵심 포인트**: 요구사항 정의 → 데이터 구조 설계 → 기능 분해 → 구현 순서 결정

---

## 📖 이론 (20분)

### 프로젝트 기획 5단계

```
1. 무엇을 만들까? → 아이디어 (한 줄 요약)
2. 누가 사용할까? → 대상 사용자
3. 어떤 기능이 필요할까? → 기능 목록 (MVP)
4. 데이터 구조를 어떻게 짤까? → DB 테이블 설계
5. 어떤 순서로 만들까? → 구현 로드맵
```

### MVP (Minimum Viable Product)
- 최소 기능 제품: 핵심 기능만 먼저 → 피드백 → 확장
- "완벽한 계획"보다 "빠른 첫 완성"이 중요!

### 프로젝트 후보 (택 1)

| 프로젝트 | 난이도 | 핵심 기능 |
|----------|--------|-----------|
| 📋 습관 트래커 | ⭐⭐ | 습관 CRUD, 매일 체크, 통계 |
| 📒 블로그/일기장 | ⭐⭐⭐ | 글쓰기, 태그, 검색, 공개/비공개 |
| 🛒 상품+장바구니 | ⭐⭐⭐ | 상품 등록, 카테고리, 장바구니 |
| 🎯 자유 주제 | 자유 | 나만의 아이디어 |

---

## 🔨 가이드 실습 (25분)

### Step 1: 프로젝트 선택 + 기획서 (10분)
```
[선택한 프로젝트]를 만들 거야.
먼저 기획서를 작성해줘:
- 프로젝트 한 줄 설명
- 대상 사용자
- 핵심 기능 목록 (MVP)
- 나중에 추가할 기능 (확장)
- 사용할 기술 스택
코드는 아직 쓰지 말고 기획만!
```

### Step 2: 데이터 구조 설계 (10분)
```
이 프로젝트의 DB 테이블을 설계해줘:
- 어떤 테이블이 필요한지
- 각 테이블의 열(column)
- 테이블 간 관계 (1:N, N:M)
- SQL CREATE TABLE 문
```

### Step 3: 구현 로드맵 (5분)
```
이 프로젝트를 어떤 순서로 만들면 좋을지
단계별 로드맵을 제안해줘. (27~28강에서 구현할 거야)
```
