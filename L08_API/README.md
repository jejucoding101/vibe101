> **핵심 포인트**: API의 개념을 이해하고, 공개 API를 활용해 실제 데이터를 가져오는 실습

---

## 📖 이론 (20분)

### API가 뭐예요?

**API**는 **Application Programming Interface**의 약자입니다.  
어렵게 들리지만, 사실 우리 일상 속에 넘쳐납니다.

> 💡 **식당 비유로 이해하기**
>
> 당신은 **손님(앱)** 이고, 주방은 **서버(데이터)** 입니다.  
> 웨이터가 주문을 주방에 전달하고 음식을 가져다주는 역할, 그게 바로 **API** 입니다.  
> 당신은 주방에 직접 들어갈 필요 없이, 웨이터(API)를 통해 원하는 것을 얻습니다.

---

### 왜 API를 쓰나요?

| 상황 | API 없이 | API 있으면 |
|------|----------|------------|
| 날씨 앱 | 직접 기상 관측소 운영 😱 | 기상청 API 호출 1줄 ✅ |
| 지도 앱 | 지도 데이터 직접 수집 😱 | Google Maps API 활용 ✅ |
| 로그인 | 보안 시스템 직접 구현 😱 | Google OAuth API 사용 ✅ |
| 번역 | AI 모델 직접 학습 😱 | DeepL API 호출 ✅ |

→ **남이 만든 강력한 기능을 내 프로그램에 붙여 쓰는 것!**

---

### API는 어떻게 동작하나요?

```
[내 프로그램]  ──── 요청(Request) ────▶  [API 서버]
               ◀─── 응답(Response) ────  [API 서버]
```

**요청(Request)**: "날씨 알려줘!" → API 주소(URL)로 보냄  
**응답(Response)**: `{ "city": "Seoul", "temp": 18, "sky": "맑음" }` → JSON 형태로 돌아옴

---

### API 주소(URL) 읽는 법

```
https://api.weather.example.com/current?city=Seoul&lang=ko
│────────────────────────────────│ │────│ │───────────────│
         API 서버 주소              경로   쿼리 파라미터(조건)
```

| 구성요소 | 역할 | 예시 |
|---------|------|------|
| **기본 주소** | API 서버의 집 주소 | `https://api.weather.com` |
| **경로(Path)** | 어떤 데이터를 원하나 | `/current` (현재 날씨) |
| **파라미터** | 세부 조건 | `city=Seoul` |

---

### 공개 API vs 비공개 API

| 종류 | 설명 | 예시 |
|------|------|------|
| **공개 API** | 누구나 무료로 사용 가능 | 한국관광공사, 공공데이터포털 |
| **키 필요 API** | 가입 후 API 키 발급 | OpenWeather, Google Maps |
| **비공개 API** | 회사 내부용 | 회사 내부 서비스 |

> 💡 **API 키**란? — 도서관 회원증처럼, "이 사람이 허가된 사용자"임을 증명하는 비밀 코드

---

### HTTP 상태 코드 — API의 신호등

| 코드 | 의미 | 일상 비유 |
|------|------|----------|
| **200** | 성공! ✅ | 주문한 음식이 나왔어요 |
| **400** | 내 요청이 잘못됨 ❌ | 메뉴에 없는 음식을 주문했어요 |
| **401** | 인증 필요 🔒 | 회원권이 없어요 |
| **404** | 찾을 수 없음 🔍 | 그 메뉴는 오늘 없어요 |
| **500** | 서버 오류 🔥 | 주방에 불이 났어요 |

## 🔨 가이드 실습 (25분)

### 실습 1: 첫 번째 API 호출 — 공공 API 맛보기 (10분)

아래 프롬프트를 AI에게 그대로 붙여넣으세요:

```
[역할] API 초보자 튜터
[목표] 다음 공개 API를 호출해서 데이터를 가져오는 프로그램 작성
[API] https://jsonplaceholder.typicode.com/posts/1
      (가짜 블로그 포스트 데이터를 반환하는 연습용 무료 API)
[요구사항]
- JavaScript(Node.js)와 Python 두 버전 모두 작성
- API에서 받은 데이터를 콘솔에 예쁘게 출력
- 각 줄마다 한국어 주석으로 설명 추가
- 초보자도 이해할 수 있도록 각 단계를 설명하는 출력 메시지 포함
```

---

### 실습 2: 날씨 API 호출하기 (10분)

```
[역할] API 실습 전문가
[목표] Open-Meteo API(무료, 키 불필요)로 서울 날씨를 가져오는 프로그램
[API 정보]
- 주소: https://api.open-meteo.com/v1/forecast
- 파라미터: latitude=37.5665&longitude=126.9780&current_weather=true
[요구사항]
- 현재 기온(temperature), 바람 속도(windspeed) 출력
- 날씨 코드(weathercode)를 한국어로 번역해서 출력
  (0=맑음, 1=대체로 맑음, 2=부분 흐림, 3=흐림, 61=비, 71=눈)
- 오류가 발생하면 친절한 에러 메시지 출력
- JavaScript 버전으로 작성
```

---

### 실습 3: API 응답을 예쁘게 출력하기 (5분)

```
아래 API를 호출해서 결과를 표(테이블) 형태로 콘솔에 출력해줘:
https://jsonplaceholder.typicode.com/users

각 사용자의 이름(name), 이메일(email), 도시(address.city)만 추출해서
번호를 붙여 깔끔하게 출력해줘. JavaScript로 작성해줘.
```
