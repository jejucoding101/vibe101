## 📖 이론 (20분)

### JSON이란?

**J**ava**S**cript **O**bject **N**otation — 데이터를 주고받는 **세계 표준** 형식입니다.

```json
{
  "이름": "홍길동",
  "나이": 25,
  "취미": ["독서", "코딩", "등산"],
  "주소": {
    "도시": "서울",
    "구": "강남구"
  },
  "결혼여부": false
}
```

### JSON 규칙 5가지

1. **키(key)**는 반드시 큰따옴표(`"`)로 감쌈
2. **값(value)**: 문자열, 숫자, 배열(`[]`), 객체(`{}`), true/false, null
3. **배열**: 여러 값의 목록 `["사과", "바나나", "포도"]`
4. **중첩**: 객체 안에 객체, 배열 안에 객체 가능
5. **쉼표**: 마지막 항목 뒤에 쉼표 ❌ (CSV와 다름!)

### CSV vs JSON

| | CSV | JSON |
|---|-----|------|
| 형태 | 표(2차원) | 트리(다차원) |
| 적합 | 단순한 목록 | 복잡한 구조 데이터 |
| 가독성 | 엑셀로 열기 좋음 | 사람/기계 모두 읽기 좋음 |
| 용도 | 데이터 내보내기 | API 통신, 설정 파일 |

### 실습 1: JSON 설계 + 생성 (8분)

```
나의 프로필을 JSON 파일로 만들어줘:
- 이름, 나이, 직업
- 취미 (배열로 3개 이상)
- 좋아하는 음식 (배열)
- SNS 계정 (객체: 플랫폼명-아이디 쌍)

```
