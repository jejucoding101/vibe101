### 자동화의 수준

| 수준 | 설명 | 예시 |
|------|------|------|
| **수동** | 매번 직접 실행 | `node script.js` |
| **반자동** | 한 번 실행하면 알아서 | 파일 감시 → 자동 처리 |
| **완전 자동** | 정해진 시간/조건에 자동 | 매일 오전 9시 보고서 생성 |

### 스케줄링

```python
# Python — schedule 라이브러리
import schedule
schedule.every().day.at("09:00").do(generate_report)
schedule.every(30).minutes.do(check_new_files)
```

```javascript
// Node.js — node-cron
const cron = require('node-cron');
cron.schedule('0 9 * * *', () => generateReport());  // 매일 오전 9시
```

## 🔨 가이드 실습 (25분)

### 실습 1: 파일 감시 + 자동 처리 (10분)
```
특정 폴더를 감시하다가 새 파일이 추가되면
자동으로 날짜별 폴더로 분류하는 프로그램을 만들어줘.
Node.js(chokidar) 또는 Python(watchdog) 사용.
```

### 실습 2: 정기 보고서 자동 생성 (10분)
```
매시간(또는 특정 시간) 자동으로:
1. 프로젝트 폴더의 파일 통계 수집
2. 마크다운 보고서 자동 생성
3. 이전 보고서와 비교하여 변경점 기록
```

### 실습 3: 졸업 프로젝트 돌아보기 (5분)
```
지금까지 만든 프로젝트의:

2. 총 코드 라인 수 세기
3. 사용한 기술 스택 정리
4. README.md에 프로젝트 요약 업데이트
```
