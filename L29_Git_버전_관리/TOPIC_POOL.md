# 🎯 화두 풀 — Git & 버전 관리

---

## 🟢 입문
1. **README 작성** ⭐ — "프로젝트 소개, 스크린샷, 설치법, 사용법이 포함된 멋진 README"
2. **.gitignore 최적화** — "프로젝트 언어/환경에 맞는 .gitignore 생성"
3. **커밋 메시지 규칙** — "feat:, fix:, docs: 등 컨벤셔널 커밋 실습"
4. **git log 분석** — "커밋 이력을 보기 좋게 표시하는 다양한 옵션"
5. **이전 상태 복원** — "특정 커밋으로 되돌리기 (git checkout, git revert)"

## 🟡 도전
6. **브랜치 실험** ⭐ — "새 기능을 브랜치에서 개발 → 완성 후 main에 병합"
7. **충돌 해결** — "같은 파일을 두 브랜치에서 수정 → 충돌 해결 연습"
8. **태그 & 릴리즈** — "v1.0 태그 달기, GitHub Releases 사용"
9. **이슈 관리** — "GitHub Issues로 할일/버그 관리"
10. **PR(Pull Request)** — "브랜치에서 작업 → PR 생성 → 코드 리뷰 → 머지"

## 🔴 심화
11. **GitHub Actions 맛보기** — "push 할 때 자동으로 테스트 실행하는 CI"
12. **오픈소스 둘러보기** — "유명 오픈소스의 Git 이력, 이슈, PR 구경하기"

## 💡 실생활
13. **일상 문서 버전 관리** — "마크다운으로 문서 작성 + Git으로 변경이력 관리"
14. **포트폴리오 GitHub Pages** — "GitHub Pages로 포트폴리오 정적 사이트 호스팅"

---

## 🤔 생각해볼 점
- Git 없이 코딩하는 것은 저장 없이 문서를 작성하는 것과 같다
- 커밋 단위를 잘게 나눌수록 나중에 문제를 찾기 쉽다
