# 📌 5강: 오류와 친해지기 — 에러는 적이 아니라 단서다

> **핵심 포인트**: 에러 메시지의 구조를 읽는 법과 AI를 활용한 디버깅 워크플로우

---

## 📖 이론 (20분)

### 에러는 왜 발생할까?

프로그램이 실행되다가 **예상하지 못한 상황**을 만나면 에러가 발생합니다. 에러는 프로그램이 "여기서 문제가 있어요!"라고 알려주는 **신호**입니다.

### 에러의 3가지 종류

| 종류 | 언제 발생 | 예시 |
|------|----------|------|
| **문법 에러** (Syntax Error) | 코드를 읽는 단계 | 괄호 안 닫음, 쉼표 빠짐 |
| **실행 에러** (Runtime Error) | 코드를 실행하는 중 | 파일이 없음, 0으로 나누기 |
| **논리 에러** (Logic Error) | 결과가 틀림 | 평균 구하는데 합만 출력 |

### 에러 메시지 읽는 법

```
// JavaScript 에러 예시
ReferenceError: myVariable is not defined
    at Object.<anonymous> (calc.js:15:1)
```

```
# Python 에러 예시
Traceback (most recent call last):
  File "calc.py", line 15, in <module>
NameError: name 'myVariable' is not defined
```

에러 메시지의 3가지 정보:
1. **에러 종류**: `ReferenceError`, `NameError` → 무슨 문제인지
2. **에러 설명**: `is not defined` → 왜 발생했는지
3. **위치 정보**: `calc.js:15` → 어디서 발생했는지

### AI에게 디버깅 요청하는 법

```
이 코드를 실행하니 다음 에러가 나와:

[에러 메시지 전체를 그대로 복사 붙여넣기]

원인을 분석하고 수정해줘.
```

> 💡 **핵심**: 에러 메시지를 **있는 그대로** 붙여넣으세요. AI가 가장 정확하게 분석할 수 있습니다.

## 🔨 가이드 실습 (25분)

### 실습 1: 일부러 에러 만들기 (10분)

Antigravity에게 요청:
```
JavaScript와 Python으로 에러가 나는 코드 예제를 각각 3개씩 만들어줘:
1. 문법 에러 (Syntax Error)
2. 실행 에러 (Runtime Error)  
3. 논리 에러 (Logic Error)

각각 어디서 에러가 나는지 주석으로 표시하되, 수정 방법은 아직 알려주지 말고.
```

직접 실행해보고 에러 메시지를 관찰하세요. 그 다음:

```
위 에러들의 원인과 수정 방법을 설명해줘.
```

### 실습 2: 에러 해결 릴레이 (10분)

```
간단한 계산기 프로그램을 만들어줘.
단, 다음 3가지 버그를 일부러 넣어줘 (수정은 하지 말고):
1. 변수명 오타
2. 0으로 나눌 때 처리 없음
3. 결과 계산 로직이 틀림

어떤 버그가 있는지는 따로 알려주지 마.
```

직접 실행하고, 에러를 하나씩 찾아서 AI에게 수정 요청해보세요.

### 실습 3: 에러 예방하기 (5분)

```
방금 수정한 계산기를 더 견고하게 만들어줘:
- 숫자가 아닌 입력에 대한 처리
- 너무 큰 숫자 처리
- 사용자에게 친절한 에러 메시지 표시
```
