# 🎯 화두 풀 — 프롬프트 엔지니어링

> 프롬프트를 다양하게 실험하며 AI와의 소통 능력을 키워보세요.

---

## 🟢 입문 (바로 시도 가능)

### 1. 같은 프로그램 5가지 프롬프트로 요청하기
> 간단한 프로그램(예: 가위바위보)을 5가지 다른 방식으로 요청하고 결과 비교
- 한 줄 요청 / 3줄 요청 / R.G.C 패턴 / 예시 포함 / 코드 스타일 지정

### 2. 역할(Role) 바꾸기 실험
> 같은 요청을 "초보 개발자", "시니어 엔지니어", "보안 전문가" 역할로 해보기
- 코드 복잡도, 보안 처리, 주석 양이 어떻게 달라지나?

### 3. 한국어 vs 영어 프롬프트
> 같은 요청을 한국어와 영어로 해보고 결과 비교
- AI가 영어 프롬프트에 더 잘 반응하는 경우가 있나?

### 4. "~처럼 만들어줘" 패턴
> "네이버 검색창처럼 동작하는 입력 필드" / "카카오톡 채팅처럼 보이는 UI"
- 유명 서비스를 몰로 삼으면 AI가 레퍼런스를 이해

### 5. 주석 달기 요청
> 이미 있는 코드에 "초보자가 이해할 수 있게 한글 주석 달아줘"

---

## 🟡 도전 (조금 더 생각 필요)

### 6. 단계적 지시 vs 한번에 지시 ⭐ 추천
> 복잡한 프로그램을 (A) 한 번에 전체 요청 vs (B) 기능 하나씩 추가 요청
- 어떤 방식이 더 좋은 결과를 내는지 비교

### 7. 코드 스타일 지정 ⭐ 추천
> "변수명은 한글로 해줘" / "함수를 많이 나눠줘" / "한 함수가 10줄을 넘지 않게"
- 프롬프트로 코드 스타일을 얼마나 제어할 수 있나?

### 8. 에러 처리 깊이 요청
> "모든 입력에 대해 유효성 검사를 해줘" vs "기본적인 에러 처리만"
- 결과 코드의 견고함이 어떻게 달라지는지 비교

### 9. 출력 형식 지정
> "결과를 표로 보여줘" / "JSON으로 출력해줘" / "마크다운으로 정리해줘"

### 10. 제약 조건 실험
> "외부 라이브러리 없이" / "10줄 이내로" / "함수 하나도 쓰지 말고"

---

## 🔴 심화 (배운 것 조합 필요)

### 11. 리팩토링 요청
> 먼저 간단하게 만든 코드를 "더 깔끔하게 리팩토링해줘"라고 요청
- 어떤 부분이 바뀌는지, 왜 바뀌는지 AI에게 설명을 요청

### 12. 테스트 코드 요청
> "이 프로그램이 잘 동작하는지 확인할 수 있는 테스트를 만들어줘"

### 13. 문서화 요청
> "이 프로그램의 사용법을 README.md로 작성해줘"

---

## 💡 실생활 응용

### 14. "비개발자 동료에게 설명하듯 만들어줘"
> 프롬프트에 설명 대상을 명시하면 주석과 변수명이 달라짐

### 15. 업무 자동화 요청 패턴
> "이 엑셀 작업을 자동화하고 싶어. [구체적 작업 설명]을 프로그램으로 만들어줘"

## 🤔 생각해볼 점

- 프롬프트가 길다고 반드시 좋은 결과가 나올까?
- AI가 "잘 모르겠습니다"라고 할 때는 어떻게 프롬프트를 수정하면 좋을까?
- 프롬프트 작성 실력과 프로그래밍 실력은 같은 것일까?
